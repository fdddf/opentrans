# 多语言管理系统落地 TODO 计划

## 阶段 1：权限与租户隔离
- 添加用户全局角色（admin/user），实现全局管理员与普通用户区分
- 完善中间件：校验 JWT 后注入角色，禁止非管理员访问管理端口
- 端到端租户隔离：所有查询/更新按用户或 app 成员过滤，补全缺失的 owner 检查

## 阶段 2：订阅与用量管控
- 为翻译/创建 app 等关键接口增加订阅校验中间件
- 翻译执行前检查额度（MaxTranslations），完成后写入使用量
- Stripe webhook/回调：根据事件更新订阅状态、额度、到期时间

## 阶段 3：App 管理与来源标记
- App 模型增加来源字段（manual/synced），删除接口仅允许 manual
- 支持同步/手动添加标记，防止重复 bundleId（按租户隔离）
- App 用户角色维度：owner/admin/editor/viewer 维持，确保权限校验一致

## 阶段 4：多语言与翻译流程
- 语言清单与校验：维护受支持语言表，上传/翻译/新增语言时校验
- 翻译状态模型：按语言与全局的进度统计接口（待翻译/已翻译/需审核）
- 单条编辑、逐语言翻译、全量翻译接口梳理，统一返回状态
- 导出接口确保应用最新翻译状态并携带 meta 信息

## 阶段 5：App Meta 多语言
- AppLocalization 扩展/校验所有 meta 字段（名称/关键词/简介/推广语等）
- 语言管理：新增/移除目标语言端点，确保删除时数据安全
- 同步到 Apple Connect：按 locale 覆盖或增量更新

## 阶段 6：安全与审计
- 请求体验证/速率限制（登录、上传、翻译等高频接口）
- 审计日志：登录/翻译/订阅变更写入 UserActivity（DI 获取 DB）
- 文件上传校验（类型/大小）、翻译输入清洗，避免注入风险

## 阶段 7：前后端对齐
- Web UI 路由与菜单：基于角色/订阅控制可见性
- 新增翻译状态视图、语言管理视图、订阅状态提示
- 调整 API 调用：适配新中间件与状态字段，补全错误提示
